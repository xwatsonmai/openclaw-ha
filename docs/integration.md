# OpenClaw 集成说明

## 目标

让 OpenClaw Agent 在回答高频家庭状态问题时，优先使用 `openclaw-ha` 生成的本地感知缓存，而不是每次都现场直连 Home Assistant。

## 推荐读取顺序

### 第一优先级：快速回答
适合问题：
- 家里现在什么情况
- 哪些灯开着
- 空调开没开
- 有没有异常

优先读取：
- `data/answer_brief.json`
- `data/answer_card.md`

### 第二优先级：结构化追问
适合问题：
- 客厅空调现在什么状态
- 重点设备里谁离线了
- 晾衣架是不是开着
- 书房温度多少

优先读取：
- `data/summary.json`

### 第三优先级：实时查询 HA
仅在这些场景触发：
- 用户明确要求“实时”“马上刷新”“现在立刻查”
- 需要执行设备控制
- 缓存已过期
- 高风险状态确认（如门锁、安防、电源）
- 缓存内没有答案

## 建议决策规则

1. 先判断是不是“状态问答”
2. 如果是高频状态问答，先读缓存
3. 如果缓存时间在 fresh_seconds 内，直接回答
4. 如果处于 stale_seconds 内，可回答并附一句“基于最近同步”
5. 如果超过 stale_seconds，先刷新再答，或明确告诉用户正在按实时状态查询

## 当前缓存文件职责

- `raw_states.json`：原始状态，便于排查
- `summary.json`：结构化回答数据
- `answer_brief.json`：简版回答素材
- `answer_card.md`：文本速览卡

## 不建议的做法

- 不要把动态状态直接写进默认提示词
- 不要每次状态问答都无条件直连 HA
- 不要把控制类问题误判为缓存回答场景

## 推荐集成方式

当前更推荐：
- 保持 `openclaw-ha` 为独立项目
- 由 OpenClaw 的 `home-assistant` skill 学会如何使用它

也就是：
- `openclaw-ha` 负责提供能力层
- skill 负责指导 agent 何时读缓存、何时刷新、何时回退实时查询
